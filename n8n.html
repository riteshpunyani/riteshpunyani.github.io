The Journey

From Problem to Solution

An engineering journey that turned a recurring infrastructure limitation into a permanently free automation system.

01
❌
The Initial Problem
First attempt used a cloud hosting platform with a free PostgreSQL database — but only for one month. Once it expired, all workflows disappeared, credentials were erased, and the entire automation system lost its memory. Had to rebuild from scratch every time.
02
💡
The Core Insight
n8n itself wasn't the issue. The real dependency was the database layer. n8n stores everything inside a database — workflows, credentials, execution logs, configurations. Without a persistent database, the system behaves like temporary memory that resets repeatedly.
03
✅
The Solution Engineered
Oracle Cloud Free Tier provides always-free virtual machines — unlike most providers that offer only temporary trials. By hosting PostgreSQL on an Oracle VM, the database layer became permanently persistent, making the entire automation system free forever.

Technical Setup

Three Core Components

The complete technical implementation — replicable by anyone with basic cloud knowledge.

Component 01
☁️
Oracle Cloud Free Tier VM
Oracle provides always-free compute instances that run continuously without expiry. I used this VM to host the PostgreSQL database server — giving the system a permanent, stable home at zero cost.
Component 02
🗄️
PostgreSQL Database
Stores all n8n data — workflow definitions, API credentials, execution logs, and automation configurations. This is the persistent backbone that makes the system survive restarts and redeployments.
Component 03
⚙️
n8n Connected to VM Database
The automation engine connects to the database via environment variables. Once connected, all data is permanently stored in the Oracle VM.
DB_TYPE=postgresdb
DB_POSTGRESDB_HOST=<oracle_vm_ip>
DB_POSTGRESDB_PORT=5432
DB_POSTGRESDB_DATABASE=n8n
DB_POSTGRESDB_USER=<username>
DB_POSTGRESDB_PASSWORD=<password>
