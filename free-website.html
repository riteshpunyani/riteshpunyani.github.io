Option 01

GitHub Pages

The most powerful free hosting option — full control over design, code, and content. This very website runs on GitHub Pages.

🐙
GitHub Pages
pages.github.com
GitHub Pages turns any public repository into a live website. Upload HTML, CSS, and JavaScript files — GitHub serves them at yourusername.github.io instantly. No server, no database, no hosting bill.
  • 01Create a free account at github.com
  • 02Create a new repository named yourusername.github.io
  • 03Use AI (Claude, ChatGPT) to generate your HTML pages
  • 04Upload files via the GitHub web interface (no coding tools needed)
  • 05Go to Settings → Pages → select main branch → Save
  • 06Your site is live at yourusername.github.io within minutes
🤖 AI Tip: Paste this prompt into Claude or ChatGPT: "Build me a professional personal website page in a single HTML file with dark navy theme, my name [Name], role [Role], and sections for About, Experience, and Contact." — You get a complete page in seconds, ready to upload.
✓ Advantages
  • Complete design freedom — full HTML/CSS/JS control
  • Custom domain support (connect your own .com)
  • Free SSL (HTTPS) included automatically
  • Version history — every change tracked, easy to undo
  • AI can write all the code for you — zero coding skills needed
  • No ads, no branding from GitHub on your site
  • Industry-standard hosting used by millions of professionals
  • Unlimited bandwidth on public repos
✗ Disadvantages
  • Slight learning curve — GitHub interface is unfamiliar at first
  • Static only — no databases, login systems, or server-side code
  • Dynamic features (forms, search) need third-party services
  • Repo must be public for free Pages hosting
  • Updates require re-uploading files (no in-browser editor)
Option 02

Google Sites

The easiest way to build a website — no code, no uploads, just drag-and-drop. Part of your existing Google account.

🌐
Google Sites
sites.google.com
Google Sites is a free, no-code website builder included with every Google account. Build by inserting sections, text blocks, images, and Google Drive embeds. Live instantly at sites.google.com/view/yoursite.
  • 01Go to sites.google.com and sign in with your Google account
  • 02Click the + button to create a new site
  • 03Choose a theme and start adding sections visually
  • 04Use AI to write your content, paste it into text blocks
  • 05Embed Google Docs, Sheets, YouTube, Maps, and Calendar directly
  • 06Click Publish — your site is instantly live
🤖 AI Tip: Ask AI: "Write professional About Me, Experience, and Skills sections for a website. I am a [Your Role] with [X] years of experience in [field]." Copy-paste the result directly into Google Sites text blocks.
✓ Advantages
  • Zero technical knowledge required — fully visual editor
  • No files to manage — everything saved in Google Drive
  • Seamless Google embeds — Docs, Sheets, Forms, Maps, Calendar
  • Live in under 30 minutes from scratch
  • Free forever with any Google account
  • Mobile-responsive by default
  • Easy collaboration — share editing access with teammates
✗ Disadvantages
  • Very limited design customisation — locked to Google's templates
  • Default URL is long and unprofessional (sites.google.com/view/...)
  • No custom CSS or JavaScript allowed
  • Looks generic — hard to stand out from other Google Sites
  • SEO is poor — Google Sites ranks lower than custom domains
  • No analytics or traffic tracking built-in
  • Not suitable for a professional portfolio or personal brand
Option 03

Free Domain + Cloudflare Redirect

Get a real custom domain for free and point it to your GitHub Pages or Google Sites — making your site look fully professional.

Setup Steps
1
Get a Free Domain
Go to ua.domains or freenom.com and register a free .ua, .tk, .ml, or .ga domain. Search for your name — e.g. riteshpunyani.ua — and register for free.
riteshpunyani.ua → free forever
2
Add Domain to Cloudflare
Create a free Cloudflare account at cloudflare.com. Click Add a Site, enter your domain, and select the Free plan.
3
Update Nameservers
Cloudflare gives you two nameserver addresses. Go back to your domain registrar and replace the existing nameservers with Cloudflare's.
ns1.cloudflare.com
ns2.cloudflare.com
4
Add DNS CNAME Record
In Cloudflare DNS settings, add a CNAME record pointing your domain to your GitHub Pages URL.
CNAME @ → yourusername.github.io
5
Set Custom Domain in GitHub
In your GitHub repo go to Settings → Pages → Custom domain, enter your free domain, and save. GitHub will verify and issue a free SSL certificate automatically.
6
Done — Live in 24 Hours
DNS propagation takes up to 24 hours. Once live, your GitHub Pages site is accessible via your custom domain — with Cloudflare CDN, free SSL, and DDoS protection.
riteshpunyani.ua → live 🎉
⛅
Cloudflare Free
cloudflare.com
Cloudflare acts as a middleman between your domain and your website. It speeds up your site (global CDN), protects it (DDoS, bots), and provides free SSL. The free plan is genuinely powerful.
✓ Advantages
  • Real custom domain — looks fully professional
  • Free SSL (HTTPS) via Cloudflare
  • Global CDN — faster load times worldwide
  • DDoS protection and bot filtering
  • Works with GitHub Pages, Google Sites, or any host
  • Free email forwarding via Cloudflare Email Routing
✗ Disadvantages
  • Free TLDs (.ua, .tk) have lower trust than .com or .in
  • DNS propagation takes up to 24 hours
  • Setup involves nameservers and DNS — intimidating for beginners
  • Free domain providers occasionally discontinue free TLDs
