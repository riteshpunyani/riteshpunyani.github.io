01
Dataset Building & Feature Engineering
Pandas • NumPy • Scikit-learn preprocessing

Raw transaction logs were joined with user-account data, merchant category codes, and historical behaviour summaries. Engineered features capture both static risk attributes and dynamic behavioural signals that simple rule engines miss.

Core feature construction pipeline:

import pandas as pd
import numpy as np
from sklearn.preprocessing import StandardScaler

# Load and merge transaction + user data
txn  = pd.read_parquet('transactions.parquet')
usr  = pd.read_csv('user_profiles.csv')
df   = txn.merge(usr, on='user_id', how='left')

# Temporal features
df['hour']        = pd.to_datetime(df['ts']).dt.hour
df['is_night']    = df['hour'].between(22, 5).astype(int)
df['is_weekend']  = pd.to_datetime(df['ts']).dt.dayofweek >= 5

# Velocity features (30-min rolling window)
df = df.sort_values(['user_id', 'ts'])
df['txn_count_30m'] = (
    df.groupby('user_id')['amount']
    .transform(lambda x: x.rolling('30min').count())
)
df['amt_z_user']    = (
    df.groupby('user_id')['amount']
    .transform(lambda x: (x - x.mean()) / (x.std() + 1e-9))
)

# Device / geo mismatch flags
df['new_device']   = (df['device_id'] != df['last_device']).astype(int)
df['geo_jump']     = df['dist_km_from_last'] > 200

FEATURES = [
    'amount', 'amt_z_user', 'txn_count_30m',
    'hour', 'is_night', 'is_weekend',
    'new_device', 'geo_jump',
    'mcc_risk_score', 'account_age_days',
    'avg_txn_amt_90d', 'chargeback_rate_90d'
]
X = df[FEATURES].fillna(0)
y = df['is_fraud']

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

Random Forest Classifier & ROC Analysis
Scikit-learn • Cross-validation • AUC-ROC 0.957

A tuned Random Forest was trained using stratified 5-fold cross-validation. Threshold selection was optimised for recall at ≥85% to minimise missed fraud rather than purely maximising accuracy.

from sklearn.model_selection import StratifiedKFold

# ROC analysis

# Optimal threshold: max recall >= 0.85

03
Isolation Forest & Composite Fraud Score
Unsupervised anomaly detection • 0–100 risk score

An Isolation Forest was trained on unlabelled transaction data to capture novel fraud patterns not represented in the labelled training set. Its anomaly score was blended with the Random Forest probability to create a single interpretable 0–100 fraud-likelihood score.

from sklearn.ensemble import IsolationForest

# Unsupervised anomaly scoring
iso = IsolationForest(
    n_estimators=300,
    contamination=0.018,  # approx fraud rate
    random_state=42,
    n_jobs=-1
)
iso.fit(X_scaled)

# Raw anomaly score: more negative = more anomalous
iso_raw  = iso.decision_function(X_scaled)
# Normalise to 0-1 (1 = most anomalous)
iso_norm = 1 - (iso_raw - iso_raw.min()) / (iso_raw.max() - iso_raw.min())

# RF probability on full dataset
rf_prob  = rf.predict_proba(X_scaled)[:, 1]

# Composite fraud score (weighted blend)
ALPHA = 0.65  # supervised weight
composite = ALPHA * rf_prob + (1 - ALPHA) * iso_norm

# Scale to 0-100
df['fraud_score'] = (composite * 100).round(1)

# Risk tiers
df['risk_tier'] = pd.cut(
    df['fraud_score'],
    bins=[0, 30, 65, 100],
    labels=['LOW', 'MEDIUM', 'HIGH']
)
print(df['risk_tier'].value_counts())
